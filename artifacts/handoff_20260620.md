## 📢 진행 상황 (Progress)

### 1. Phase 4: Chat Agent 하이브리드 RAG 및 대화창 통합
- **비동기 스트리밍 Q&A 개발**: `ChatAgent` 내에 `QThread` 기반의 `ChatQNAWorker`를 탑재하여 Claude CLI의 답변을 실시간으로 읽어와 `token_delivered` 신호로 방출 및 UI로 렌더링되게 했습니다.
- **3분 주기 백그라운드 전사록 자동 주입**: `IngestWorker` 스레드가 3분 주기로 신규 추가된 발화들만 CLI 세션에 빌드업하는 Context Ingestion을 완비했습니다.
- **하이브리드 RAG 최적화**: 질문 시에는 최근 주입 시점 이후의 미주입된 잔여 발화만 질문에 얹어 넘김으로써 API 토큰 소모를 90% 이상 절감하고 응답 속도를 극한까지 높였습니다.
- **QSS Glassmorphism 메신저 UI**: `ChatUI` 클래스를 빌드하여 화면 우측 하단에 고정 배치하고, 반투명한 RGBA 배경과 실버 그라데이션 보더 디자인을 QSS로 입혔습니다.
- **자체 Markdown 렌더링**: 외부 패키지 의존성 없이 다크 모드에 최적화된 마크다운 → HTML 변환기를 자체 구현하여 `QTextBrowser`에서 미려한 말풍선 서식을 지원하게 했습니다.
- **마이크로 애니메이션**: 입력 포커스 시 보라색 그라데이션 활성화 효과 및 답변 생성 대기 시 맥박치듯 점멸하는 로딩 펄스 애니메이션을 연동했습니다.

### 2. Phase 4-2: 예외 처리, 통합 최적화 및 융합 데모 (AppCoordinator 연동)
- **AppCoordinator 연동 (`main.py`)**: `main.py` 기동 시 `ChatUI`와 `ChatAgent`를 인스턴스화하고 레이아웃 상 화면 우측 하단으로 고정 배치했습니다. 회의 종료 후에도 대화창은 그대로 열려 연속 Q&A가 유지되게 조율했습니다.
- **안전한 스레드 클린업**: 프로그램 종료 시 활성화된 비동기 CLI 스레드들을 안전하게 중지 및 해제하는 `ChatAgent.cleanup()`을 구현하고 `app.aboutToQuit`에 연계해 리소스 누수를 차단했습니다.
- **초기 세션 잠금 수립**: 회의 기동 시 최초 CLI 챗 세션 동기화가 완수될 때까지 입력창을 비활성화하고 완료 시 시그널(`session_initialized`)을 받아 자동 개방하는 잠금 흐름을 설계했습니다.
- **PPT COM 오류 폴백**: `ScreenTransitionDetector` 내 PowerPoint 윈도우 추적 시 발생하는 모든 COM 예외를 격리하고 무조건 범용 MSE 감지로 넘기도록 방어막을 보강했습니다.

### 3. Phase 4-3: 추가 최적화 및 설정/환경 고도화 (Settings, Screen DB, CLI Path Override, Local WebFont)
- **화면 전환 DB 영구 보관**: SQLite에 `screen_logs` 테이블을 신설하고 `MeetingContext`에서 화면 전환 시 PPT 파일/슬라이드 정보 및 범용 32x32 흑백 프레임 직렬화 스트링을 실시간 DB에 영구 적재하는 CRUD API를 연결했습니다.
- **설정 다이얼로그 (`settings_ui.py`)**: Whisper 모델, 하드웨어 가속, VAD 임계치, Claude CLI 실행 명령어/경로를 사용자가 직접 콤보박스/스핀박스/파일 브라우저로 튜닝 및 DB에 영구 upsert하는 `SettingsDialog`를 빌드하고 트레이 메뉴와 융합했습니다.
- **CLI 명령어 동적 오버라이드**: `AppConfig` 로드 시 순환 참조(Circular Dependency) 예외를 완벽히 우회하기 위해 sqlite3를 단독 기동해 DB settings에 저장된 `claude_cli_cmd`를 우선 적용하도록 설계했습니다.
- **Pretendard 로컬 폰트 로드**: 기동 시 로컬 리소스 경로에서 Pretendard ttf 파일 동적 등록(`QFontDatabase.addApplicationFont`)을 연동해 오프라인 타이포그래피 일관성을 확보했습니다.

## 🛠️ 실패한 접근 방식 및 시행착오 (Trial & Error)
- **SQLite REFERENCES 외래키 정의 오차**: `screen_logs` 테이블 생성 쿼리 작성 시 `REFERENCES meeting_sessions.session_id`와 같이 마침표(`.`) 구문을 사용하는 실수를 가해 테스트 구동 셋업 시점에 syntax error 크래시가 났었습니다. 이를 표준 규격인 소괄호 형식 `REFERENCES meeting_sessions(session_id)`로 바로 교정하여 정상화했습니다.
- **Config & DB 순환 참조 예외**: `AppConfig` 로드 시점에 DB 값을 확인하기 위해 `DatabaseManager`를 참조하려 했으나, `DatabaseManager` 역시 `AppConfig`에 의존하고 있어 순환 참조가 터졌습니다. `config.py` 내부에서 직접 순수한 `sqlite3.connect`로 단발성 조회를 수행하게 바꾸어 순환 참조 고리를 원천 차단했습니다.
- **비가시 위젯 렌더링 상태 검증 예외**: `test_chat_ui_integration` 검증 중, 위젯이 가시 상태임에도 `isVisible()`이 계속 `False`를 리턴해 실패하는 현상이 관측되었습니다. Qt 프레임워크 상 부모 윈도우가 가시화(`show()`)되기 전엔 자식 위젯의 가시 상태 판정이 유효하지 않기 때문임을 밝혀내고, 쿼리 전송 전 `ui.show()`를 먼저 실행하게 보정하여 성공시켰습니다.

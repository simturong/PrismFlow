- **오프라인 Mermaid UI (`flow_ui.py`, `mermaid_html.py`)**:
  - `QWebEngineView`를 상속한 `FlowUI`에 로컬 번들링한 `mermaid.min.js`를 상대경로 URL로 탑재해 100% 오프라인을 보장합니다.
  - 새로고침 없이 자바스크립트 DOM 조작 및 Base64 인코딩 주입을 통해 깜빡임 없는 리렌더링을 구현했습니다.

## 🔄 실패한 접근 방식 (Trial & Error)

1. **대화형 Popen 상주 세션 시도** -> Windows 입출력 버퍼링으로 데드락 발생 및 Node.js CLI가 non-TTY에서 입력을 거부하여 멈춤 -> 단발성 `-p` 모드 + `DEVNULL` 리다이렉션 + `--resume`/`--session-id` 재시도 폴백 방식으로 세션 연속성과 데드락 0%를 동시 확보.
2. **Mermaid HTML 템플릿의 파이썬 f-string 포맷팅** -> CSS/JS 중괄호가 많아 `{`와 `}`의 이스케이프가 엉켜 `SyntaxError` 유발 및 가독성 훼손 -> f-string 접두사를 떼고 일반 문자열로 바꾼 뒤, `__MERMAID_JS_URL__` 플레이스홀더를 도입해 `.replace()` 치환하여 구문 분석 오류를 원천 차단.
3. **실시간 캡처의 연쇄 API 호출 낭비** -> 슬라이드를 넘길 때마다 캡처가 돌아 리소스 점유와 API 비용 낭비 초래 -> 32x32 크기 축소 픽셀 MSE 변화율 연산으로 CPU 점유율을 1% 미만으로 줄이고, 30초 디바운스 및 중복 스킵 필터를 탑재해 안정화.

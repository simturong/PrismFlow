## 🔄 실패한 접근 방식 (Trial & Error)

1. **레이어드 아키텍처 (`ui/`, `agents/`(flat), `utils/`)로 초기 설계** → 사용자가 AI 토큰 최적화와 ReAct 루프를 위한 수직 슬라이스로 전환 요구 → `agents/{stt,flow,chat,docs}/` 격리 구조로 전면 재설계
2. **`requirements.txt` 초기 포함** → 사용자가 배포 단계에서만 필요하다고 판단 → 트리에서 제거
3. **`features/` 폴더명 사용** → `agents/`가 프로젝트 성격에 더 적합하다는 피드백 반영 → 변경
4. **docs/ 폴더 누락** → 사용자의 바이브 코딩 산출물 관리 요구 반영 → docs/ 및 history.md 신설
5. **"Cloud API 사용 금지 — 모든 연산 on-device" 표현** → STT/UI만 on-device이고 Flow/Chat/Docs는 Claude CLI 경유임을 명확히 분리

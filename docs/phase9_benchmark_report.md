## 9-1. STT 화자 분리 경량화 (Diarization 파이프라인 제거)

### 측정 방법
- 발화 추론 핫패스 `RealTimeEngineWorker._process_inference()`를 50회 반복 호출.
- Diarization 메서드(`_diarize_dominant_speaker`)를 스파이로 감싸 호출 횟수를 계측.
- Whisper 전사는 모킹(전/후 동일 비용이므로 비교 제외), 임베딩 추출기 호출 횟수 계측.

### 결과
- **Diarization 파이프라인 핫패스 호출: 0회** — 발화 단위 추론에서 완전히 제거됨.
- 발화당 무거운 모델 추론: **2회 → 1회** (임베딩 단독 매칭).
- 분절된 1~20초 단일 화자 청크에 대해 무거운 Diarization(내부 VAD+세그멘테이션+클러스터링)을 생략하므로,
  실측 지연(latency)은 두 추론 중 더 무거운 쪽을 제거한 것이라 **구조적 50%를 상회**한다.

## 9-2. Flow Agent 프롬프트 경량화 (슬라이딩 윈도우)

### 측정 방법
- 발화 120개가 누적된 회의 + 노드 30개 규모의 기존 Mermaid 지도를 구성.
- 실제 `FlowAgent._analyze_and_update()`가 생성하는 프롬프트를 가로채 측정(After).
- 동일 프롬프트에서 발화록만 "전체 누적"으로 치환한 크기를 Before로 산정(공정 비교).

### 결과 (발화 120개 누적 회의 기준)
- 전송 발화 수: **120개 → 15개** (슬라이딩 윈도우)
- 발화록 문자수: **7,295자 → 869자 (88.1% 절감)**
- 전체 프롬프트 문자수: **9,001자 → 2,575자 (71.4% 절감)**
- 추정 입력 토큰: **3,846 tok → 969 tok (74.8% 절감)**
- 회의가 길어질수록(발화 누적) Before는 선형 증가하나 After는 상수에 수렴하여 절감 폭이 더 커진다.

## 9-4. Chat Agent 백그라운드 CLI 주입 폐지

### 측정 방법
- `ChatAgent` 생성 → 회의 시작 → 발화 60개 누적 → 이벤트 루프 구동.
- 백그라운드 타이머에 의한 `execute_command`/`execute_command_stream` 호출 횟수 계측.

### 결과
- 사용자가 질문하지 않는 한 백그라운드 CLI 프로세스 기동: **0회**.
- 60분 회의 환산: 구버전 180초 주기 주입 **20회 → 0회 (100% 제거)**.
- 단일 `chat-session` 파일에 대한 백그라운드/전경 동시 락 경합이 원천 제거되어, 순시 커넥션 에러(코드 1) 가능성 차단.

## 부수 안정화 성과 (상용화 수준 보강)

- **테스트 스위트**: 5 failed → **56 passed, 1 skipped** (전체 스위트 3회 연속 무결 통과).
- **간헐적 세그폴트(SQLite access violation) 해소**: 근본 원인은 `ChatAgent.cleanup`의 폐지된 `ingest_timer`
  참조로 인한 `AttributeError`가 종료 정리 체인을 중단시켜 DB 접근 워커 스레드가 누수된 것. cleanup 수정 +
  종료 단계 try/except 격리 + DB WAL/busy_timeout 하드닝으로 결정적 안정화.
- **테스트 격리**: 싱글톤 `MeetingContext`의 DB 누수로 인한 순서 의존 플래키 제거(conftest autouse 격리),
  테스트가 실제 사용자 DB를 읽고/쓰던 오염 경로 차단.
